基本用語集

リポジトリ Repository

ファイルやフォルダの変更履歴をまとめて管理する場所です。プロジェクトの「保存箱」のようなイメージで、誰がいつどんな変更を加えたかをすべて記録しています。GitHubではリポジトリをオンライン上で公開・共有することができます。Google Driveとの決定的な違いは「変更の意図と履歴が構造化されて残る」点です。

コミット Commit

ファイルへの変更内容をリポジトリに記録する操作です。変更のスナップショットを残すイメージで、「いつ・何を・なぜ変更したか」をメッセージとともに保存します。コミットを積み重ねることで、作業の歴史を追うことができます。

プッシュ Push

ローカル（自分のパソコン）でのコミットを、GitHubなどのリモートリポジトリに送信する操作です。プッシュすることで、自分の変更をチームメンバーと共有したり、オンライン上にバックアップしたりできます。

プルリクエスト Pull Request

「この変更を取り込んでほしい」という提案書です。自分のブランチで行った変更をメインブランチへ統合してほしいと依頼でき、チームメンバーが内容をレビュー・コメントしたうえで承認→マージの流れで確定させます。変更を直接反映するのではなく「お願い」として提出することで、ミスを防ぎ共同作業を安全に進められます。

ブランチ Branch

メインの作業履歴から分岐して、独立した作業ができる仕組みです。新機能の開発やバグ修正を別のブランチで行うことで、メインのコードに影響を与えずに作業を進められます。作業が完了したらプルリクエストを使って統合（Merge）します。

マージ Merge

プルリクエストで提案された変更を、正式にメインブランチへ取り込む操作です。「Merge pull request」ボタンを押すことで確定され、変更が本番のコードに反映されます。最終的な承認・実行は人間が行う設計になっており、誤った変更が混入しないための重要なステップです。

はじめての Tips

GitHubとClaude Codeを繋ぐと何ができる？
GitHubは「変更履歴つきのコード専用Google Drive」のようなもの。Claude Codeと接続することで、日本語の指示だけでファイル操作・公開までが完結します。インストール不要でブラウザから使えます。
📝

README.md がプロジェクトの「顔」

リポジトリを開いたとき最初に表示されるファイルです。プロジェクトの概要を書いておくと、見た人に伝わりやすくなります。

🌐

GitHub Pages で即公開

Settings → Pages → Branch: main → Save の設定だけで、リポジトリ内の HTML ファイルがそのまま Web ページとして公開されます。

🤖

Claude Code への指示の書き方

「ファイル名＋内容＋デザイン要件」を日本語で書くだけで OK。具体的であるほど精度が上がります。

✅

人間が最終承認する設計

Claude Code はプルリクエストを作成するまでが役割。「Merge pull request」ボタンを押して確定するのは人間です。

WebページをGitHub Pagesで公開する最短手順

再現可能なステップ

  1. GitHubにリポジトリを新規作成（Add a README file にチェック必須）
  2. GitHub Settings → Applications で Claude アプリに書き込み権限を付与
  3. claude.ai/code でリポジトリを選択し、日本語で指示を送る
  4. Claude Code が Pull Request を作成 → Create pull request → Merge pull request で確定
  5. GitHub Pages（Settings → Pages → Branch: main → Save）で URL が発行され、Web ページとして公開される

注意点

⚠️ やってはいけないこと・忘れがちな設定

  • 個人情報・APIキー・パスワードはリポジトリに絶対に入れない。 Public リポジトリは世界中から閲覧可能です。
  • リポジトリ作成時に 「Add a README file」のチェックを忘れると、Claude Code が接続できない場合があります。
  • Public リポジトリに上げたファイルは 世界中から検索・閲覧可能です。公開して問題ないか確認してからプッシュしましょう。
