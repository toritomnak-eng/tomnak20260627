Git・GitHub 用語集

はじめての方のための基本用語まとめ

リポジトリ Repository

ファイルやフォルダの変更履歴をまとめて管理する場所です。プロジェクトの「保存箱」のようなイメージで、誰がいつどんな変更を加えたかをすべて記録しています。GitHubではリポジトリをオンライン上で公開・共有することができます。

コミット Commit

ファイルへの変更内容をリポジトリに記録する操作です。変更のスナップショットを残すイメージで、「いつ・何を・なぜ変更したか」をメッセージとともに保存します。コミットを積み重ねることで、作業の歴史を追うことができます。

プッシュ Push

ローカル（自分のパソコン）でのコミットを、GitHubなどのリモートリポジトリに送信する操作です。プッシュすることで、自分の変更をチームメンバーと共有したり、オンライン上にバックアップしたりできます。

プルリクエスト Pull Request

自分が加えた変更を、別のブランチ（多くはメインブランチ）に取り込んでほしいと依頼する機能です。GitHubでは変更内容を一覧表示でき、チームメンバーがコードをレビューしてコメントや承認をしてから統合する流れが作れます。

ブランチ Branch

メインの作業履歴から分岐して、独立した作業ができる仕組みです。新機能の開発やバグ修正を別のブランチで行うことで、メインのコードに影響を与えずに作業を進められます。作業が完了したらプルリクエストを使って統合します。
